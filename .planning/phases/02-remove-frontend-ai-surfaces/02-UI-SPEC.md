# Phase 02 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Ant Design 5 existing components only |
| Icon library | `@ant-design/icons` existing set only |
| Font | existing system sans-serif stack from `frontend/src/index.css` |

Phase constraint: this is a cleanup phase. Do not introduce shadcn, Tailwind, new tokens, or a visual redesign. Preserve the current Ant Design information architecture and light-shell layout while removing AI-only surfaces.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline tag gaps, compact metadata spacing |
| sm | 8px | Small button/icon gaps, compact stack spacing |
| md | 16px | Default card padding, form control spacing, row gutters |
| lg | 24px | Page content padding, header padding, section spacing |
| xl | 32px | Login card header spacing, major internal separation |
| 2xl | 48px | Large breathing room inside empty/loading areas |
| 3xl | 64px | Full-height shell anchors such as sider brand block |

Exceptions: preserve existing Ant Design control heights where already present; keep primary submit/touch targets at 44px minimum when a form uses a single dominant action, as in the login button.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 16px | 600 | 1.25 |
| Display | 18px | 600 | 1.2 |

Type contract: limit this phase to the existing system text rhythm. Use only regular (400) and semibold (600). Remove AI copy blocks instead of replacing them with new emphasis styles.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f0f2f5` | App shell background, page canvas, neutral layout field |
| Secondary (30%) | `#ffffff` | Header, sider, cards, tables, modal bodies |
| Accent (10%) | `#1677ff` | Primary CTA buttons, selected navigation item, key non-destructive links |
| Destructive | `#ff4d4f` | High-severity alert emphasis, destructive confirmations, error feedback |

Accent reserved for: login submit, search action, selected menu state, primary operational buttons that remain after AI removal, and non-destructive path links such as "查看全部". Do not repurpose accent to fill the visual gap left by removed AI controls.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 确认告警 |
| Empty state heading | 暂无活跃告警 |
| Empty state body | 系统运行正常。若需排查历史记录，请前往“告警管理”。 |
| Error state | 操作失败，请重试；若持续失败，请检查网络连接或重新登录。 |
| Destructive confirmation | 无新增破坏性操作：本阶段仅移除 AI 入口与字段，不新增用户可见的删除或清空流程。 |

Product naming contract for this phase: all retained visible labels in touched screens must use “告警系统” or the specific module name only. Remove “AI” from menu labels, page titles, modal titles, helper text, and product title wherever this phase touches them.

---

## Interaction Contract

- Remove the `/ai` route, its sidebar menu item, related icon usage, and `AIAssistant` page exports without leaving empty navigation slots or broken redirects.
- Keep the current authenticated shell intact: same sider/header/content structure, same navigation order, same light theme, with the AI item simply removed.
- Dashboard active-alert cards must retain only operational actions that still exist after backend AI removal: `确认` and `静默`. Do not replace the removed AI action with a new workflow.
- Visual hierarchy on the dashboard remains anchored by the active-alert section: after AI removal, the user's eye should land first on connection warnings and P0/high-severity alerts, then on the active-alert cards/list, with charts and secondary statistics remaining supporting context.
- Dashboard modal behavior must not be repurposed for analysis. Remove the AI modal entirely; the default alert interaction becomes list review plus direct operational actions.
- Alert list expanded rows must continue to show operational data such as message and labels only. Remove AI summary/root-cause sections instead of substituting speculative text.
- When AI-derived fields are absent, no placeholder, badge, chip, or empty container should remain. The layout should collapse cleanly with existing card/table spacing.
- Preserve current feedback patterns: success via Ant Design `message.success`, failures via `message.error`, loading via existing `Spin`/table loading states, and empty states via Ant Design `Alert`.
- Preserve status semantics already established in the product: severity colors, status tags, and warning/success messaging remain unchanged.
