<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→`GET /api/v1/deliveries*` | 用户可操控 query 参数，前端只能消费受保护只读接口 |
| alert UI→delivery history UI | 从告警页传递的 `alert_id` deeplink 不能伪装成恢复授权 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | I | `frontend/src/pages/Deliveries.tsx` | mitigate | 只展示后端已返回的快照字段，不在前端展开 channel `config`、secret 或任何未下发敏感字段。 |
| T-19-02 | T | `frontend/src/pages/Deliveries.tsx` query parsing | mitigate | 只构造后端允许的过滤参数集，时间范围统一转 RFC3339，分页边界遵循现有 GET 合同。 |
| T-19-03 | E | `/deliveries` route | mitigate | `/deliveries` 继续受 `capabilityViewConfig` 保护，本 plan 不引入任何恢复 POST 入口，避免 viewer 经由历史页获得写能力。 |
| T-19-04 | R | alert→delivery deeplink | mitigate | 使用显式 `alert_id` query 跳转并在页面中回显当前筛选条件，让操作者能证明查看的是哪条告警的投递历史。 |
</threat_model>
