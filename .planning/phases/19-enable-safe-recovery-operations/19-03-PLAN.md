<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser modal→recovery POST API | 操作者填写的 reason 文本进入后端审计与真实恢复动作 |
| delivery history UI→operator permissions | 同一页面同时承载 viewer 只读与 operator 可写能力，容易发生 UI 越权暴露 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-10 | E | `frontend/src/pages/Deliveries.tsx` action rendering | mitigate | 仅在 `canProcessAlerts(user)` 且 `delivery_status === 'failed'` 时渲染恢复按钮，viewer 永远只读。 |
| T-19-11 | R | recovery reason capture | mitigate | reason 作为必填字段提交并在成功/失败结果回显中显示 recovery id/result，避免“点了但无记录”争议。 |
| T-19-12 | D | duplicate clicks | mitigate | 单条 delivery 的恢复请求中禁用按钮并维护 per-row loading，降低并发双发风险。 |
| T-19-13 | T | stale UI after recovery | mitigate | 成功提交后强制刷新列表和详情，展示 resulting delivery id 或 error_message，避免用户基于旧状态继续操作。 |
</threat_model>
