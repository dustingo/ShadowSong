<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated operator → delivery read API | 运维用户通过 JWT/capability 读取账本，若授权缺口存在会泄露通知内容与失败证据 |
| handler query params → delivery service query | 列表筛选参数来自不可信请求，错误解析或宽查询可能造成越权枚举或资源消耗 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-07 | I | `/api/v1/deliveries*` | mitigate (blocking) | 两个接口都挂在现有 `JWTAuth + RequireCapability(CapabilityViewConfig)` 下，测试必须覆盖 401/403/200 三类响应。 |
| T-18-08 | T | `internal/handlers/delivery.go` filter parsing | mitigate | 只接受固定 query 参数，`limit/offset` 做边界约束，所有查询走 service/GORM，不拼接原始 SQL。 |
| T-18-09 | D | `GET /api/v1/deliveries` | mitigate | 列表接口限制分页参数并维持最小查询面，不在本 phase 扩成无界全文搜索或复杂聚合。 |
</threat_model>
